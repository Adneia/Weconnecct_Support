<analysis>**original_problem_statement:**
O usuário solicitou a criação de um sistema de controle de chamados (tickets) para a equipe de atendimento da WeConnect, chamado ELO. O sistema precisa gerenciar fluxos de trabalho detalhados, uma grande base de dados de pedidos, e automatizar mudanças de status.

**PRODUCT REQUIREMENTS:**
-   **Telas e Funcionalidades:**
    -   : Visão geral e estatísticas.
    -   : Formulário para criar/editar tickets com busca de pedidos (incluindo por Galpão + Nota), auto-preenchimento, e lógica para impedir duplicados. Opção de Encerrar ao criar.
    -   : Tabela com filtros avançados e exportação de relatórios customizados (Ag. Compras, Ag. Logística).
    -   : Upload de Excel/CSV com processamento em background.
    -   : Tela para CRUD de textos padrão, acessível a todos, com um log de alterações visível para o admin.
-   **Integrações e Automações:**
    -   Sincronização de atendimentos com a planilha Google Sheets Atendimentos 2026_E.
    -   Sincronização de chamados de devolução com uma planilha Google Sheets separada (Devoluções).
    -   Ao criar/editar um chamado de devolução, o sistema deve registrar o status (Aguardando, Estornado, Reenviado).
    -   Exibir o estoque disponível de uma planilha externa.
-   **Autenticação:** JWT (email/senha).

**User's preferred language**: Portuguese (pt-BR)

**what currently exists?**
Uma aplicação full-stack (React + FastAPI + MongoDB) que gerencia chamados de atendimento. O sistema foi significativamente expandido nesta sessão com as seguintes funcionalidades:
- **Gestão de Textos Padrão:** A funcionalidade foi massivamente expandida com dezenas de novos textos para diversas categorias (Comprovante de Entrega, Falha Fornecedor) e status (Aguardando, Entregue, Ag. Logística), com botões correspondentes na UI. A lógica de substituição de placeholders foi corrigida.

    - O campo de anotação se tornou opcional na edição de chamados, com um pop-up de confirmação.
    - O Dashboard teve a coluna Total substituída por uma coluna Próximo Dia Útil que projeta os chamados em aberto.
    - A tela de login agora possui um botão para mostrar/ocultar a senha.
    - Um indicador visual (ELO - Ambiente de Teste) foi adicionado ao ambiente de preview para diferenciá-lo da produção.
- **Automações e Correções:**
    - A lógica de sincronização com o Google Sheets foi corrigida para encontrar e atualizar chamados corretamente, mesmo quando o número do pedido é alterado.
    - A regra de negócio que define o Motivo de Pendência foi aprimorada para classificar corretamente status em maiúsculas como Enviado (com exceção de ENTREGUE).
    - Links de rastreio desatualizados da Total Express foram substituídos.
- **Exportação:** O arquivo Excel exportado da lista de chamados agora inclui as colunas Motivo Pendência e Status Pedido.
